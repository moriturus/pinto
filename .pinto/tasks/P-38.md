# **Summary**

`pinto doctor` inspects the complete board twice on every run: once before and once after the fix stage, even without `--fix` or when no safe fix was applied. The second inspection doubles I/O and parsing for the common healthy-board case.

# **Acceptance Criteria**

- [ ] `doctor` without `--fix` performs exactly one full inspection.
- [ ] `doctor --fix` re-inspects only when at least one fix was applied.
- [ ] A regression test covers the single-inspection path and the existing doctor tests pass.
