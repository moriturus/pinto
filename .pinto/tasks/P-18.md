# **Summary**

Parse Markdown Acceptance Criteria checkboxes and expose their completion progress without changing the plain-text item format.

# **Implementation Plan**

## Domain parser

Add a pure Markdown task-list parser in the backlog domain. Count unordered and ordered task-list markers, including nested and quoted items, while ignoring fenced code blocks. Represent the result as completed over total and derive it from the existing body at read time so persistence remains unchanged.

## Display surfaces

Add the computed progress to `show` and the Kanban details popup. Add an opt-in `--acceptance-criteria` / `-A` column to `list --long` and `board --long`, reusing the same formatter and progress value. Keep the existing detail-field alignment stable by excluding the longer new label from the shared padding baseline.

## Completion warning

When a move enters the configured done column, preserve the successful transition and emit a warning only when the item has unchecked task-list boxes. Reuse the outcome for CLI and Kanban transitions so both paths have the same non-blocking behavior.

## Verification and compatibility

Drive the parser, formatter, service, CLI, and Kanban behavior with tests first. Verify the body remains byte-for-byte unchanged through CLI edits and moves, add a runnable demo under `demos/`, update user documentation, dogfood the feature with `cargo run`, then run the repository quality gate.

# **Acceptance Criteria**

- [x] PBI details and the Kanban details popup can display Acceptance Criteria progress as completed over total checkboxes.
- [x] `list --long` and `board --long` can include the same progress as an opt-in column.
- [x] Moving a PBI to the configured done column emits a non-blocking warning when its Acceptance Criteria contain unchecked boxes.
- [x] Checkbox counting leaves the stored Markdown body and existing item data format unchanged.
