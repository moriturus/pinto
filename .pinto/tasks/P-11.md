Summary

Add an assignee filter to list and board so users can focus on PBIs assigned to a particular person.

Implementation Approach

- Add an optional exact-match --assignee <NAME> value to list and board, with -u as its short form. Keep -a for the existing --all-labels option.
- Carry the filter through ListFilter and BoardQuery, applying it conjunctively before list hierarchy and board column grouping so JSON and existing ordering are preserved.
- Follow TDD: write failing domain/service and CLI integration tests before the production changes, then cover composition with status, Sprint, label, and search filters, JSON output, and unchanged unfiltered ordering.
- Update the CLI guide and a focused demo dataset and README, then verify the behavior with cargo run.

Acceptance Criteria

- [x] pinto list --assignee <name> returns only PBIs whose assignee matches the requested name.
- [x] pinto board --assignee <name> applies the same filter to every displayed workflow column.
- [x] The filter composes with existing status, Sprint, label, and search filters and works with JSON output.
- [x] Omitting the filter preserves the current results and ordering.
