# **Summary**

Define a deliberate evolution policy for public Rust errors, structs, enums, and re-exports.

# **Acceptance Criteria**

- [ ] Public error types have an extension policy that prevents routine additions from unexpectedly breaking exhaustive matches.
- [ ] The compatibility impact of changing that policy is recorded before the next minor release.
- [ ] Public re-exports are audited and unnecessary symbols are reduced without hiding APIs required by the CLI boundary.
- [ ] Public input types have a builder or documented field-extension strategy before the 1.0 release.
