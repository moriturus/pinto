# **Summary**

Provide a read-only `pinto export --json` command that returns one machine-readable snapshot containing the board's PBIs, Sprints, configuration, and Definition of Done.

## Implementation Plan

1. Add an `export` top-level CLI subcommand with `--json`; keep it read-only and load one coherent board snapshot through the service layer.
2. Add a service snapshot operation that reads active PBIs, Sprints, effective loaded configuration, and optional common DoD from the configured backend without writes or lock acquisition.
3. Reuse existing list/sprint JSON field mappings so IDs, field meanings, and UTC RFC3339 timestamps remain consistent. Emit a top-level object with `items`, `sprints`, `config`, and `dod`.
4. Follow Red → Green → Refactor with CLI integration coverage for populated and empty/optional data, plus a no-mutation check.
5. Document the command in README/docs and add a focused demo board under `demos/single/export`.

# **Acceptance Criteria**

- [x] `pinto export --json` returns PBIs, Sprints, board configuration, and the shared Definition of Done in one JSON document.
- [x] The export uses the same field meanings and UTC timestamp format as the existing JSON commands.
- [x] Running the export does not modify `.pinto` data or require a server.
