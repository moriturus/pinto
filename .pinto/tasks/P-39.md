# **Summary**

`rm --force` is irreversible and file-backend boards keep no history, so recovering from a mistaken mutation falls back to manual Git commands at best. Provide a guided undo for the most recent completed board mutation, at minimum on the Git backend.

# **Acceptance Criteria**

- [ ] A feature decision record covers scope, per-backend behavior, and compatibility impact.
- [ ] On the Git backend, one documented command reverts the most recent completed board mutation.
- [ ] On backends without history, the command fails with an actionable message describing the available recovery options.
