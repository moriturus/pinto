# **Summary**

Add integration tests that validate the persisted demo boards under `demos/` against the current pinto behavior, so changes to the specification that make a supported demo incompatible fail the standard quality gate without modifying repository data.

# **Acceptance Criteria**

- [x] Integration tests cover every supported normal-use demo board under `demos/`.
- [x] The tests exercise each covered board through appropriate non-mutating pinto CLI operations and identify the affected demo when validation fails.
- [x] Every demo that intentionally reproduces parse or command errors is verified against its expected exit code and diagnostic output; no demo is silently excluded.
- [x] All test-induced state changes run in isolated temporary copies outside the repository, and the test run leaves the checked-in demo data and Git worktree unchanged.
- [x] The tests run under `cargo test --all-features --locked` and therefore execute as part of `mise run check`.
