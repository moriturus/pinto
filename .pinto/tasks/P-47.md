# **Summary**

Make configurable Kanban modified-key bindings work reliably across terminal event encodings, including the documented `Ctrl+?` regex-search shortcut, other `Ctrl+` combinations, and bindings using other supported modifiers. The fix and verification should cover the shared key-matching path rather than treating one shortcut or modifier as an isolated case.

# **Implementation Plan**

- Inventory how crossterm reports representative `Ctrl`, `Alt`, `Cmd/Super`, `Meta`, `Hyper`, and `Shift` combinations, including `Ctrl+?`, and define the supported normalization/matching behavior.
- Apply the behavior consistently to default and user-configured Kanban key bindings while keeping `?` reserved for help.
- Add focused unit and PTY regression coverage for `Ctrl+?`, other modified-key combinations, and unaffected plain-key behavior.

# **Acceptance Criteria**

- [ ] `Ctrl+?` opens regex search from the documented default shortcut in a standard terminal.
- [ ] Other configured `Ctrl+` key bindings are recognized through the same Kanban input path in a standard terminal.
- [ ] Configured bindings using other supported modifiers are recognized through the same Kanban input path in a standard terminal.
- [ ] Automated regression coverage exercises `Ctrl+?`, other `Ctrl+` combinations, and other supported modifiers, including the terminal event representations under test.
- [ ] `?` still opens help and existing plain-key bindings keep their current behavior.
