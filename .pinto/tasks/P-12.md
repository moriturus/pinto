# **Summary**

Add a stale-item filter that finds PBIs whose `updated` timestamp has not changed for a requested duration.

# **Implementation Plan**

## Duration Syntax and Query Semantics

- Add `list --stale <duration>` with a lightweight single-value syntax: a positive integer followed by `s`, `m`, `h`, `d`, or `w` (for example, `7d`).
- Capture the current UTC time once at the CLI boundary and pass an `updated` cutoff into the list filter; an item is stale when its `updated` timestamp is at or before that cutoff.
- Reject zero, negative, malformed, unsupported-unit, and overflowing durations with actionable input guidance.

## Filtering and Compatibility

- Extend the existing list-filter matching path so stale matching composes conjunctively with status, Sprint, label, search, roots-only, archived, and every existing output mode.
- Keep the filter read-only, preserve canonical backlog ordering, and leave the item JSON schema unchanged.

## Verification and Documentation

- Add parser and service tests plus CLI integration coverage for matching, boundary behavior, invalid input, composed filters, JSON output, and unchanged timestamps.
- Document the syntax in the CLI help, README, mdBook CLI guide, and a runnable stale-filter demo dataset.

# **Acceptance Criteria**

- [x] `pinto list --stale <duration>` returns PBIs older than the duration at the time of the query.
- [x] A documented duration such as `7d` is accepted, and invalid durations produce an actionable user error.
- [x] The filter composes with the existing status, Sprint, label, search, and output options.
- [x] The stale query is read-only and does not update item timestamps.
