# **Summary**

Keep pinto lightweight while preserving SQLite as a supported, opt-in storage backend.

# **Acceptance Criteria**

- [ ] SQLite support remains available through the non-default `sqlite` feature.
- [ ] The default build does not enable the SQLite dependency, and the feature boundary remains documented.
- [ ] Plain-text and Git-diff guarantees are stated for file and Git backends, with SQLite described as an explicit exception.
- [ ] New commands, backends, and reports require a Scrum-related need, an explanation of why existing functionality is insufficient, and a record of dependency, persistence, and migration impact.
- [ ] Future SQLite schema work improves migration and compatibility guidance without making SQLite support or its non-default status a removal target.
