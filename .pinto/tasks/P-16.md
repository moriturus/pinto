# **Summary**

Add `pinto doctor` to diagnose board integrity problems and offer only safe, explicit repairs for recoverable cases.

# **Acceptance Criteria**

- [ ] `pinto doctor` detects dangling dependency, parent, and Sprint references; relation cycles; duplicate IDs; issued-ID inconsistencies; invalid statuses; rank anomalies; and tasks/archive collisions.
- [ ] Each reported issue identifies the relevant path or PBI and gives an actionable repair direction.
- [ ] The command returns a non-zero exit status when integrity problems are found and a zero status for a clean board.
- [ ] `pinto doctor --fix` applies only safe mechanical repairs, reports every change, and never silently discards board data.
