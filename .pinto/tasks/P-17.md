# **Summary**

Add a read-only `pinto next` command that identifies the highest-ranked PBIs that are ready to start.

# **Acceptance Criteria**

- [ ] `pinto next` returns the highest-ranked unstarted PBI whose dependencies are all in the configured done column.
- [ ] `pinto next -n <count>` returns up to the requested number of eligible PBIs in rank order.
- [ ] `pinto next --sprint <id>` restricts candidates to the requested Sprint.
- [ ] Blocked or already-completed PBIs are not returned, and the command does not modify the board.
