# **Summary**

Align release metadata with the actual publication state and make SQLite schema incompatibility visible before release.

# **Acceptance Criteria**

- [ ] Unreleased version metadata remains clearly unreleased until the public release is made, and published version references stay consistent.
- [ ] The SQLite schema v1-to-v2 breaking change documents affected users, symptoms, backup requirements, and downgrade or recovery guidance.
- [ ] A release check detects mismatches among package versions, lockfiles, installation examples, changelog headings, and tags.
- [ ] Public release is blocked until the breaking-change documentation and version checks are complete.
