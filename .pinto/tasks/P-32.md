# **Summary**

Reduce maintenance cost in large modules and documentation tests while preserving behavior and useful TUI coverage.

# **Acceptance Criteria**

- [ ] Large CLI and TUI modules are split incrementally along clear responsibilities without changing behavior.
- [ ] Link targets, TOML, JSON, YAML, and Markdown structure are checked structurally where possible.
- [ ] Executable documentation examples are verified through demos or CLI behavior tests, and brittle prose-only assertions are reduced over time.
- [ ] TUI smoke tests cover startup, exit, re-entry, and representative interaction paths without duplicating every key operation.
