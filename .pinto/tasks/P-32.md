# **Summary**
Reduce maintenance cost in large modules and documentation tests while preserving behavior and useful TUI coverage.

# **Implementation Plan**
## Responsibility boundaries
- Move automation-plan execution from the large CLI command module into a dedicated child module without changing command behavior.
- Move pure Kanban input decisions and terminal lifecycle guards into dedicated runtime modules while preserving the existing event loop and rendering behavior.

## Structural documentation checks
- Parse maintained TOML and JSON documents with their existing structural parsers.
- Parse workflow YAML with a test-only YAML parser and validate Markdown links with a code-aware structural scanner.
- Keep prose checks only for user-facing wording that cannot be expressed as a structural invariant.

## Executable docs and TUI smoke coverage
- Execute the documentation demo flow in an integration test on an isolated temporary board.
- Keep PTY coverage for startup, exit, resize/editor interaction, details interaction, filters, and repeated shell re-entry, while sharing lifecycle assertions and transport helpers.

## Verification
- Follow Red → Green → Refactor for new structural and executable checks.
- Run targeted tests, mise run check, and the relevant cargo run commands before closing this PBI.

# **Acceptance Criteria**
- [x] Large CLI and TUI modules are split incrementally along clear responsibilities without changing behavior.
- [x] Link targets, TOML, JSON, YAML, and Markdown structure are checked structurally where possible.
- [x] Executable documentation examples are verified through demos or CLI behavior tests, and brittle prose-only assertions are reduced over time.
- [x] TUI smoke tests cover startup, exit, re-entry, and representative interaction paths without duplicating every key operation.
