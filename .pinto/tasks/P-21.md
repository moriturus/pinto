# **Summary**

Allow label-setting CLI options to accept multiple label values after one flag, including `add --label foo bar` and `edit <id> --label foo bar`, and apply the same behavior to any other option that mutates labels.

# **Acceptance Criteria**

- [ ] `pinto add "Example" --label foo bar` stores both `foo` and `bar` on the new PBI.
- [ ] `pinto edit <id> --label foo bar` replaces the PBI's labels with both `foo` and `bar`.
- [ ] Every other existing CLI option that sets or replaces labels, if any, accepts the same variable-number syntax and receives aligned help and test coverage.
- [ ] Existing repeated `--label` usage and label-filtering options remain compatible and are not changed into label-setting operations.
- [ ] CLI tests and user-facing examples document the supported multi-label syntax.
