# **Summary**

Four in-process unit tests in the `pinto` binary crate assert English output but resolve messages through the process-global `pinto::i18n::current()` localizer, which is initialized from the parent `LC_ALL`/`LANG`. Under a Japanese parent (`LC_ALL=ja_JP.UTF-8`) they fail:

- `cli::commands::tests::automation_results_extract_created_ids_and_sanitize_errors`
- `cli::commands::tests::sprint_capacity_rejects_an_incomplete_programmatic_argument_set`
- `cli::kanban::tests::render::popup_on_empty_column_still_shows_a_placeholder`
- `cli::kanban::tests::render::render_help_overlay_includes_secondary_operations_and_clear_filter`

These are not integration tests that spawn processes, so the shared spawn-helper locale pin from P-44 does not reach them. The `#[cfg(test)]` trick does not help either: the binary crate links the library (`pinto`) as a non-test dependency, so `current()` still reads the real environment. Fixing them cleanly means giving the tested production paths (`automation_execution_result`, the `cmd_sprint` capacity path, and the kanban render pipeline) an injectable localizer instead of calling the global `current()` (~128 call sites), or introducing a deterministic test seam for the process-wide localizer.

## Implementation Plan

1. Add localizer-accepting internal seams for automation-result formatting and Sprint command execution. Keep the existing production entry points as thin wrappers that pass `pinto::i18n::current()`.
2. Thread an explicit `&Localizer` through the Kanban render pipeline and its localized helper functions. The interactive runtime continues to pass `current()`, while render tests pass a deterministic English localizer created with `localizer_from`.
3. Update the four regression tests to use the explicit English localizer and run them under `LC_ALL=ja_JP.UTF-8`, proving that parent-process locale no longer controls their English assertions.
4. Run the relevant binary tests, the full quality gate, and an iterative review. No environment mutation, global override, new dependency, persistence change, or compatibility shim will be introduced. Test-only compatibility wrappers that became unused in production are limited with `#[cfg(test)]`.
