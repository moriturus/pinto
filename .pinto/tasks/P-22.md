# **Summary**

Document and validate a reproducible local CI workflow using `nektos/act` so release build and package failures can be caught before pushing.

# **Implementation Plan**

## Host-aware runner contract

Document Docker and `nektos/act` prerequisites, job discovery with `act -l`, and host-specific job selection. On macOS/Linux, `act push -j release` runs the Linux release job without starting the Windows matrix. On Windows, `act push -j check --matrix os:windows-latest -P windows-latest=-self-hosted` selects only the Windows matrix entry and runs it on the Windows host. Explain that the Windows path is act self-hosted execution rather than a Docker guest, so it needs a real Git checkout containing `.git`, Node.js for JavaScript actions, a compatible `unzip`, and the native Windows toolchain; PowerShell script execution is enabled only for the current process when required. It does not replace GitHub-hosted runner parity.

## Release path and diagnostics

Reuse the existing `.github/workflows/ci.yml` release job without changing production CI semantics. Document that it executes `cargo build --release --all-features --locked`, `./scripts/verify-package.sh`, and the locked source install, plus the direct-command fallback and common failure diagnostics.

## Environment and documentation

Record Docker requirements for the macOS/Linux container path, native Git/Node.js/unzip/mise/Rust/rustup requirements for the Windows self-hosted path, and that the current jobs need no repository secrets. Explain safe handling of any future secrets without committing them. Add a focused mdBook/README guide.

## Scope and verification

This is an operational documentation PBI; per explicit scope, it does not add demo data or new tests. Verification uses the existing mdBook build and repository quality gate, targeted act dry-runs and host execution where permitted, direct release commands, cargo run dogfooding, iterative review, and final PBI completion only after all criteria are verified.

# **Acceptance Criteria**

- [x] The repository documents prerequisites and host-specific targeted `nektos/act` commands for running the relevant CI jobs.
- [x] The Linux release path, including `cargo build --release --all-features --locked` and `./scripts/verify-package.sh`, can be reproduced locally with actionable failure output.
- [x] macOS/Linux instructions skip the Windows matrix, while Windows instructions select the `windows-latest` matrix entry with `--matrix` and `-P windows-latest=-self-hosted`; host limitations and required environment or secret behavior are documented without changing production CI semantics.
