# **Summary**

`export --json` has no inverse: restoring a board from a snapshot or migrating from another tool requires hand-written automation plans. Add an import that recreates board data from the exported JSON contract.

# **Acceptance Criteria**

- [ ] A documented command restores PBIs, Sprints, configuration, and the shared DoD from an `export --json` snapshot.
- [ ] Importing into a non-empty board fails fast unless an explicit mode is chosen.
- [ ] A round-trip test proves that export followed by import reproduces an equivalent board.
- [ ] A feature decision record covers persistence and compatibility impact.
