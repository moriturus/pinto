# **Summary**

Extend Sprint closing so unfinished PBIs can be moved to a following Sprint or released in one operation, with spillover visible in velocity reporting.

# **Acceptance Criteria**

- [ ] `pinto sprint close <id> --rollover <target>` assigns unfinished PBIs from the closing Sprint to the target Sprint in one operation.
- [ ] `pinto sprint close <id> --release` removes the Sprint assignment from unfinished PBIs in one operation.
- [ ] Completed PBIs remain unchanged when either option is used.
- [ ] Invalid targets or conflicting close options leave the Sprint and its assignments unchanged.
- [ ] Sprint velocity reporting distinguishes unfinished spillover points for retrospective use.
