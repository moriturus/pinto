# **Summary**

Extend Sprint closing so unfinished PBIs can be moved to a following Sprint or released in one operation, with spillover visible in velocity reporting.

# **Implementation Plan**

1. Introduce an explicit Sprint-close action (`retain`, `rollover`, or `release`) and persist the actual close time plus a close-time spillover snapshot containing estimated points, item count, and unestimated item count. Validate the source transition and rollover target before the first write, hold the board write lock for the operation, and restore prior records if a record save fails. Once durable writes complete, preserve the existing Git commit-failure contract so the files remain available for manual recovery.
2. Add mutually exclusive `sprint close --rollover <target>` and `--release` CLI options. Reassign or clear only PBIs that are unfinished at close time; leave completed PBIs byte-for-byte equivalent at the domain level, including their original Sprint assignment and timestamps. A plain close retains the current assignment behavior.
3. Expose spillover as a separate retrospective value in velocity output. Count a closed Sprint's velocity only from PBIs completed no later than its actual close time, and never add spillover points to completed points, averages, or change percentages. This also prevents retained work completed later from inflating past velocity; rolled-over work contributes when it reaches Done in its target Sprint.
4. Drive the change through failing domain/service, persistence, and CLI integration tests, including invalid/conflicting options and repeated backend coverage. Update Sprint documentation and the persisted lifecycle demo, verify the flows with `cargo run`, run iterative review to convergence, and finish with `mise run check`.

# **Acceptance Criteria**

- [x] `pinto sprint close <id> --rollover <target>` assigns unfinished PBIs from the closing Sprint to the target Sprint in one operation.
- [x] `pinto sprint close <id> --release` removes the Sprint assignment from unfinished PBIs in one operation.
- [x] Completed PBIs remain unchanged when either option is used.
- [x] Invalid targets or conflicting close options leave the Sprint and its assignments unchanged.
- [x] Sprint velocity reporting distinguishes unfinished spillover points for retrospective use.
