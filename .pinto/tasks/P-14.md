# **Summary**

Add Sprint and label filters to the interactive Kanban view so the TUI can focus on the active work scope.

# **Acceptance Criteria**

- [ ] `pinto kanban --sprint <id>` displays only PBIs assigned to the requested Sprint.
- [ ] `pinto kanban --label <label>` applies the same label-matching behavior used by the board view.
- [ ] Sprint and label filters compose with `--column`, `--search`, and `--regex`.
- [ ] Startup filters do not modify board data and are described in command help and documentation.
