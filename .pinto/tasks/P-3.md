## Summary
Investigated CI run 29405214703. The macOS 26 arm64 failure was a PTY teardown timing issue in the shell lifecycle test: after two Kanban sessions the process returned to the third shell prompt, but the test's three-second final exit wait expired while Ctrl-D was being handled. The production lifecycle leak was not reproduced on local macOS, Linux, or Windows checks. The coverage failure was a metric mismatch: the LLVM text total was 94.99% while the Cobertura artifact reported 95.422%. Added regression coverage for the Windows-compatible automation-plan source error paths, made the gate validate the uploaded Cobertura line-rate directly, and kept the 95% threshold. CI push triggers now include both main and develop.
