# **Summary**

The board format is designed for Git, yet no documentation covers merging boards edited in parallel branches or clones. Sequential IDs collide, `issued_ids` produces textual conflicts, and merged boards can contain duplicate IDs without a documented recovery path.

# **Acceptance Criteria**

- [ ] The book gains a runbook explaining how parallel clones allocate conflicting IDs and how to resolve `issued_ids` and task-file conflicts.
- [ ] The runbook includes doctor-based verification steps for a merged board.
- [ ] Documentation link and snippet tests pass.
