# **Summary**

Provide a consistent JSON board export for automation without making ordinary reads block on board-wide locking.

# **Acceptance Criteria**

- [ ] `export --json` acquires the board lock before opening configuration and storage and holds it until the export completes.
- [ ] Concurrency tests prove that an export cannot combine old and new records while `sprint close` or `rebalance` is writing.
- [ ] Ordinary read commands remain non-blocking and their lack of board-wide snapshot isolation is documented.
- [ ] Shell and agent automation that requires a consistent board is directed to `export --json`.
