# **Summary**

Warn Scrum users when the points assigned to a Sprint exceed its configured capacity or the selected velocity comparison, without blocking the workflow.

# **Implementation Plan**

## Warning inputs

- Sum only estimated points on PBIs assigned to the target Sprint; unestimated PBIs do not contribute to the point total.
- Compare that total with the configured `SprintCapacity.hours` value. Capacity is a numeric planning threshold, and equality remains within the limit.
- Compare the total with the average completed points from the five most recent closed Sprints before the target Sprint. Exclude the target Sprint so its current plan cannot affect its own baseline; omit this warning when no historical comparison exists.

## Command integration

- Run the same read-only warning calculation after a successful `sprint start` and after successful single or bulk `sprint add` assignment.
- Print localized warnings to stderr while preserving the requested operation and exit status. Emit one warning per exceeded threshold and no output when all applicable thresholds are within bounds or unset.
- Keep the warning calculation in the service layer so CLI and future callers share the point aggregation and threshold rules without writing board data.

## TDD and verification

- Add pure service tests for point aggregation, equality boundaries, capacity/velocity threshold selection, and missing comparisons before implementing the behavior.
- Add CLI integration tests covering `sprint start`, single/bulk `sprint add`, non-blocking persistence, within-limit cases, and localized warning output.
- Update the Sprint CLI documentation and validate with `cargo run`, the relevant test suites, lint, formatting, and the repository check.

# **Acceptance Criteria**

- [x] `pinto sprint start` warns when the assigned point total exceeds the applicable capacity or velocity threshold.
- [x] `pinto sprint add` provides the same warning when the assignment would exceed the applicable threshold.
- [x] The warning is non-blocking and the requested valid operation still completes.
- [x] No warning is emitted when the assigned work is within the applicable threshold or no comparison is configured.
