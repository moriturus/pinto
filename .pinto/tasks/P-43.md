# **Summary**

Localized help output mixes languages: command descriptions and section headings are translated while most argument descriptions fall back to English (28 `cli-arg-*` catalog keys versus 103 `#[arg]` declarations). A partially translated screen is more confusing than an untranslated one because the translated headings signal that the screen is fully localized.

# **Acceptance Criteria**

- [x] Every `#[arg]` help string resolves from the Fluent catalog in en-US and ja-JP.
- [x] An automated test fails when localized `--help` output for a supported non-English locale contains English fallback argument text.
- [x] clap built-in strings are localized or explicitly documented as out of scope.

# Implementation Plan

## Background

The argument help in `src/cli/args.rs` is resolved by `localize_command`, which
looks up `cli-arg-{path}-{arg_id}` keys in the Fluent catalog and falls back to
the English `#[arg]` doc comment when a key is missing. The catalog currently
holds only 28 argument keys, so 97 argument descriptions render in English even
under a non-English locale.

## Approach

1. **Catalog coverage (AC1):** Add the 97 missing argument keys to both
   `en-US.ftl` and `ja-JP.ftl`. The en-US value mirrors the current doc comment
   verbatim so English output is unchanged; the ja-JP value follows the existing
   translations' vocabulary and tone. Doc comments stay as Rustdoc while the
   catalog becomes the source of truth for help text (the same dual-management
   pattern as the existing 28 keys).
2. **Fallback-detection test (AC2):** Add a test in `src/cli/args.rs` that walks
   the localized command tree and asserts every non-builtin argument resolves a
   catalog key under a non-English locale (ja-JP). A missing key means an English
   fallback, so the test fails. An integration test also asserts the ja-JP
   `--help` output contains no English fallback argument text.
3. **clap built-in strings (AC3):** Documented as out of scope. `clap` adds the
   `-h`/`--help` and `-V`/`--version` flags, the generated `help` subcommand, the
   `Usage:` prefix, and the `Possible values:` block (from `ValueEnum` variants)
   during its build phase, *after* the `mut_args` hook runs, so a shared
   `cli-arg-builtin-*` key cannot reach them. Localizing them would require
   disabling clap's automatic help/version handling and re-declaring it on every
   command, which is not worth the surface for a lightweight CLI. The exclusion is
   recorded in the `localized_command` doc comment. Every pinto-authored command
   and argument description is localized (a missing `cli-command-pinto-restore`
   key was also filled in to remove the last mixed-language command line).
