# **Summary**

Improve commit boundaries, review expectations, and maintainer guidance so ongoing maintenance remains sustainable.

# **Acceptance Criteria**

- [ ] Cross-cutting work is split into small commits that keep the repository green and separate data, service, CLI, and documentation changes where practical.
- [ ] Acceptance conditions are reviewed before large changes, and destructive or release-related changes receive review from someone other than the primary maintainer when available.
- [ ] Security response timing is described as a response target rather than a guarantee.
- [ ] Release and security responsibilities include a documented fallback when a second maintainer is unavailable.
