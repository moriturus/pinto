# **Summary**

CI run https://github.com/moriturus/pinto/actions/runs/29384691650 for commit `573233a` has two independent failures in the cross-platform `check` matrix.

## Investigation

- Ubuntu fails in `tests/cli/kanban.rs` because Clippy reports `clippy::unnecessary_mut_passed` for the `winsize` argument to `libc::openpty`.
- The failure was reproduced with Rust 1.97 in the official `rust:1.97-bookworm` Docker image using the same Clippy flags as `mise run lint`.
- `libc` 0.2.186 declares this argument as `*const winsize` on Linux and `*mut winsize` on Apple platforms, so the current cross-platform `&mut window` call is lint-clean only on macOS.
- Windows fails before the check runs. `jdx/mise-action@v4.2.0` installs only Rust and mdBook, then mise 2026.7.6 auto-installs the three missing Cargo tools during the following `rustup component add` step. Their concurrent Rustup/Cargo access conflicts on `cargo.exe`, `cargo.pdb`, and partial downloads.
- The existing step-local `MISE_TASK_RUN_AUTO_INSTALL=false` only controls task execution and is applied too late. The generic mise control for this path is `MISE_AUTO_INSTALL=false`.
- MSRV, current-stable, macOS check, release, coverage, and dependency-policy jobs are green in the same run.

## Implementation Plan

1. Make the PTY setup mirror each supported libc signature with `cfg`-gated, reference-based calls: pass `&window` on const-signature platforms and `&mut window` on mutable-signature platforms. Keep the unsafe FFI boundary localized and do not add explicit raw-pointer construction.
2. Set `MISE_AUTO_INSTALL: "false"` at the `check` job level so it applies before toolchain setup and Rust component installation. Continue installing only `rust mdbook` explicitly in that job; remove the narrower task-only setting if it is redundant after verification.
3. Pin the mise CLI `version` input used by `jdx/mise-action` to the release verified by the repaired matrix instead of resolving `latest` on every run.
4. Use the existing PTY integration coverage and CI matrix for this CI-only fix; do not add a new test or TDD cycle. Re-run the Linux Docker lint reproduction, the local release gate, and the full GitHub Actions workflow.
