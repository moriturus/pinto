# **Summary**

Expose the validated automation-plan JSON Schema through `pinto automate --schema` so agents and CI can inspect the accepted plan shape before running it.

# **Acceptance Criteria**

- [ ] `pinto automate --schema` prints a machine-readable JSON Schema without requiring an execution plan.
- [ ] The schema describes the accepted command-array structure and rejects unknown fields or unsafe command forms consistently with the validator.
- [ ] The schema documentation and validation tests remain synchronized with the supported automation plan.
- [ ] Existing `--plan`, `--dry-run`, and `--json` automation behavior remains unchanged.
