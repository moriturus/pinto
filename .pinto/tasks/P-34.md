# **Summary**

Refactor the oversized implementation and test code into appropriately scoped modules using Rust 2018 module-system conventions, while preserving existing behavior and adding no new tests.

# **Implementation Plan**

## Scope

Split the files that still mix multiple concerns after the P-32 split. Size is the symptom; the split axis is responsibility.

- `src/cli/commands.rs` — keep `dispatch`, `entrypoint`, error formatting, and shared filter/warning helpers in the parent; move command handlers into command-group children (`item`, `relations`, `sprint`, `board`, `maintenance`, `session`); move the inline `mod tests` to `commands/tests.rs`.
- `src/cli/format.rs` — move implementations into the existing `item`/`report` children (previously thin façades) and new `text` (width/truncate) and `tree` (column tree) children; move the inline `mod tests` to `format/tests.rs`; the parent keeps only module declarations and the `DEFAULT_TERM_WIDTH` re-export.
- `src/cli/kanban/runtime.rs` — keep the event loop in the parent; move board actions (submit/transition/reorder/edit/reload/search) to `runtime/actions.rs` and all rendering plus the popup/help scroll-max math to `runtime/render.rs`; move the inline test mods to `runtime/tests.rs`.
- `src/storage/sqlite_repository.rs` — keep connection and schema management in the parent; move item persistence to `sqlite_repository/items.rs` and sprint persistence to `sqlite_repository/sprints.rs`; move the inline `mod tests` to `sqlite_repository/tests.rs`.
- `src/cli/kanban/tests.rs` — promote the inner test mods to child files under `kanban/tests/` (`fixtures`, `display_scope`, `view`, `popup`, `tree`, `search_input`, `wrap`, `render`).
- `src/service/item/tests.rs` — split the tests by the implementation module they cover into `item/tests/{crud,edit,reorder}.rs`; the shared `create_sprint_for_test` helper moved to `service::test_support`.
- `tests/cli/misc.rs` — redistribute the grab-bag into six focused integration files: `usage` (help/aliases/argument errors), `init_discovery`, `backends` (file/SQLite/migrate), `git_backend`, `corruption`, and `rebalance`.

Out of scope: `service/doctor.rs`, `service/sprint.rs`, and `storage/file_repository.rs` stay as-is; each is large but already covers a single responsibility.

## Conventions

- Rust 2018 layout only: parent `foo.rs` plus `foo/` children, no `mod.rs`.
- The parent re-exports (`pub use`) whatever was public before, so external paths and behavior stay unchanged.
- Code moves verbatim; only `mod` declarations, imports, and visibility (`pub(super)`/`pub(crate)`) are adjusted.

## Incidental adjustments

- `tests/package.rs` raises its packaged-file-count guard from 100 to 130 because the split adds module files to the published package; the guard itself is unchanged.
- Intra-doc links in `sqlite_repository.rs` now name the re-export paths for the repository traits that no longer appear in the parent's imports.

## Verification

- `mise run check` (test / lint / Rust and mdBook docs / fmt) passes.
