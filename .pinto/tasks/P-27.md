# **Summary**

Keep shared contributor guidance current while allowing each developer to maintain local agent instructions.

# **Implementation Plan**

## Shared baseline

Keep `AGENTS.base.md` as the versioned, common reference and update it to describe the shipped Kanban TUI, the current test and fuzz workflows, the pinned quality gate, and the actual roles of the direct dependencies.

## Local overlays and repository hygiene

Document how a developer derives a local `AGENTS.md` overlay from the shared baseline and adds machine- or preference-specific instructions without changing the shared file. Keep root-level `AGENTS.md`, `CLAUDE.md`, and `.claude/` content ignored so personal guidance cannot enter a commit.

## Documentation consistency

Synchronize `CONTRIBUTING.md` and the mdBook contributor page with the shared-baseline model, and remove or correct stale guidance and local links. Preserve migration history as current narrative context, but do not retain links or claims about an absent frozen snapshot file.

## Regression coverage

Add documentation integration tests that assert the required shared-baseline sections, the local-ignore contract, and valid relative Markdown links across maintained contributor guidance. Run the focused tests and the full quality gate before completing the item.

# **Acceptance Criteria**

- [x] The shared baseline reflects the shipped TUI, current testing and fuzzing practices, and actual dependency roles.
- [x] The shared baseline is the common reference for people working in the project, not a claim that one local instruction file is the sole source of agent guidance.
- [x] Developers can derive and use personal agent instruction files locally from the shared baseline.
- [x] Personal agent instruction files are excluded from version control and are not committed to the repository.
- [x] Historical references and local links are either current or removed, and structural link checks cover the maintained guidance.
