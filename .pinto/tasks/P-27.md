# **Summary**

Keep shared contributor guidance current while allowing each developer to maintain local agent instructions.

# **Acceptance Criteria**

- [ ] The shared baseline reflects the shipped TUI, current testing and fuzzing practices, and actual dependency roles.
- [ ] The shared baseline is the common reference for people working in the project, not a claim that one local instruction file is the sole source of agent guidance.
- [ ] Developers can derive and use personal agent instruction files locally from the shared baseline.
- [ ] Personal agent instruction files are excluded from version control and are not committed to the repository.
- [ ] Historical references and local links are either current or removed, and structural link checks cover the maintained guidance.
