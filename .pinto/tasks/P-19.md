# **Summary**

Allow pinto commands to discover the nearest ancestor `.pinto` board when invoked from a repository subdirectory, with an explicit override for scripts.

# **Acceptance Criteria**

- [ ] Commands issued from a descendant directory locate and use the nearest ancestor `.pinto` board.
- [ ] An explicit board-directory override is available through `--dir` or `PINTO_DIR` for scripts and agents.
- [ ] Searching stops at the filesystem root or documented repository boundary and reports an actionable error when no board is found.
- [ ] Invoking pinto from the board root keeps the current behavior.
