# Security policy

## Supported versions

Security fixes are applied to the latest version on the default branch. Earlier
versions are not maintained separately.

## Reporting a vulnerability

Please do not file public issues for suspected vulnerabilities. Report them
privately to the repository maintainer through GitHub's private vulnerability
reporting feature. Include a clear description, reproduction steps, affected
versions, and any suggested mitigation.

We will acknowledge reports within seven days, assess the impact, and work with
you on a fix and coordinated disclosure where appropriate.
